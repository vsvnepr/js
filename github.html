Подсказки для Git и GitHub

Git — система контроля версий. Технология, которую можно скачать на компьютер.
Github — сервис, который позволяет работать с твоими Git проектами. Это называется репозитории. Помимо Github существуют другие сервисы (например Bitbucket, GitLab).

Качаем и устанавливаем Git
После установки, в командной строке используем git
Переходим с помощью cmd в папку нашего проекта
    Базовые команды в Windows:
  • git init - для инициализации репозитория
  • git status - показывает статус файлов, и что сейчас происходит
  • git add index.html - добавляет файл для отслеживания git
  • git add . - добавляет все файлы для отслеживания git
  • .gitignore - Создаем такой файл в проекте. Если мы хотим добавить некоторые файлы или папки в исключение для отслеживания. Пример папка /logs или файл error.txt

    • git config

    Перед выполнением коммитов нужно представиться гиту, коммандами:
  • git config --global user.name - Показывает имя пользователя
  • git config --global user.name 'new user' - Изменяет имя пользователя
  • git config --global user.email - Показывает email пользователя
  • git config --global user.email 'test@mail.ru' - Изменяет email пользователя

  • git commit -m 'nameCommit' - создает коммит с сообщением. Коммит - это фикс версии, (условно инкрементная-копия).

    • Работа с ветками

  • git branch - показывает список веток
  • git branch branchName - создает новую ветку branchName
  • git branch -D branchName - удаляет ветку branchName
  • git checkout branchName - переключиться на ветку branchName
  • git checkout -b branchName - создает и переключается на ветку branchName

    • Объединение веток

  • git merge branchName - совмещает текущую ветку с branchName

    • Работа с GitHub

    Для начала зарегистрируйтесь на GitHub
    Затем создайте репозитории
    Введите свой email и пароль от gitHub в командной строке
  • git remote add origin https://github.com/vsvnepr/name.git - привязка удаленного репозитория к локальному, прописываем адрес
  • git push -u origin master - залить локальные коммиты, на gitHub
